# bip-monitor

Триаж собственных рабочих чатов **BiP** локальной LLM, с обучением на правках владельца.

Официальный BiP Developer API личные чаты не отдаёт (там модель Service/бота), поэтому
механизм — клиентская автоматизация: attach по CDP к своему залогиненному BiP Desktop.

## Что делает

- **Триаж непрочитанного** — читает список чатов, ничего не открывая: тема, срочность,
  нужен ли ответ по каждому входящему. Строго read-only.
- **Разбор конкретного чата** — открывает чат и разбирает всю подгруженную ленту.
  Открытие помечает сообщения прочитанными, поэтому только по явной команде.
- **Обучение** — правка владельца («этот чат всегда срочный», «слово „авария“ — высокая»)
  сохраняется правилом и дальше применяется автоматически.
- **Отправка** — одно сообщение по явной команде с дословно подтверждённым текстом.
  Автоответов нет by design.

## Требования

- BiP Desktop (Electron), запущенный с `--remote-debugging-port=9222 --remote-allow-origins=*`
- локальная LLM с OpenAI-совместимым API на `http://localhost:8080/v1` (llama.cpp/Qwen)
- Python 3.10+, `pip install websocket-client`

## Быстрый старт

```powershell
Get-Process BiP* | Stop-Process -Force
& 'C:\Program Files\BiP\BiP.exe' --remote-debugging-port=9222 --remote-allow-origins=*
irm http://localhost:9222/json/version

cd skills/bip-triage/scripts
python learning_store.py init
python bip_desktop_monitor.py --once
```

Проверки: `python test_bip_triage.py`, `python bip_desktop_monitor.py --selftest`,
`python bip_send.py --selftest`.

## Приватность — читать до включения

В рабочих чатах лежат данные третьих лиц. Плагин устроен так, чтобы они не расползались:

- тексты сообщений пишутся **только** в локальный файл вне git и вне хранилища знаний;
- база обучения не хранит ни текстов, ни имён, ни номеров, ни jid — только HMAC-ссылки
  и, отдельно, ключевые слова, которые владелец сам назначил правилами;
- в консоль выводятся хеши, длины и вердикты, но не содержимое;
- анализ выполняет **локальная** модель. Вывод переписки в облачную модель — осознанное
  исключение, а не режим по умолчанию.

Риск блокировки аккаунта за автоматизацию клиента ложится на владельца — используйте
собственный или специально заведённый аккаунт.

## Что внутри

```
skills/bip-triage/
  SKILL.md                     что делать по запросу + границы
  references/
    cdp-and-client.md          селекторы, четыре особенности клиента, механика отправки
    learning-and-storage.md    формат правок, области действия, жизненный цикл правил
  scripts/
    bip_cdp.py                 тонкий CDP-клиент (websocket, без Playwright)
    bip_desktop_monitor.py     триаж: режимы list и chat
    bip_discover.py            снятие структуры DOM (без текста) при обновлении клиента
    bip_send.py                отправка одного сообщения, с проверками до Enter
    learning_store.py          store обучения
    store_schema.py            общий контракт обучения скиллов v0.2 (вендорится)
    test_bip_triage.py         проверки, включая «в базе нет ПДн»
```
